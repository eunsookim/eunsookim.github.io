# 사이트 전체 다국어(i18n) 설계

**Date:** 2026-03-11
**Status:** Approved
**Scope:** 전체 사이트 (public + admin editor)

## 요구사항

- 한국어(기본) + 영어 지원
- 서브패스 라우팅: `/ko/...`, `/en/...`
- 한국어 작성 → DeepL Free API 자동 번역 → 영어 저장
- 번역 후 수동 교정 가능 (에디터에서 ko/en 탭)
- 자동 언어 감지: 쿠키 > 브라우저 언어 > IP 국가
- 언어 전환 토글 (헤더)
- slug는 영어 기반 통일 (한국어 slug 없음, 기존 설계와 동일)

## URL 구조

```
eunsookim.dev/           → 언어 감지 후 /ko 또는 /en 리다이렉트
eunsookim.dev/ko/        → 한국어 홈
eunsookim.dev/en/        → 영어 홈
eunsookim.dev/ko/blog/my-post → 한국어 블로그 글
eunsookim.dev/en/blog/my-post → 영어 블로그 글
```

## Slug 전략

- slug는 영어 기반 단일 컬럼 유지 (기존 `posts.slug`, `projects.slug` 그대로)
- 양쪽 언어에서 동일한 slug 사용: `/ko/blog/react-guide`, `/en/blog/react-guide`
- `slug_en` 별도 컬럼 불필요 — 기존 스키마와 호환

## 언어 감지 (우선순위)

1. **쿠키** (`preferred-lang`) — 사용자가 언어 토글로 직접 선택한 값
2. **브라우저 언어** (`Accept-Language` 헤더) — ko 포함 시 ko, 아니면 en
3. **IP 국가** (`x-vercel-ip-country` 헤더) — KR이면 ko, 그 외 en
4. **기본값** — ko

> 브라우저 언어를 IP보다 우선하는 이유: 해외 거주 한국인이 브라우저를 한국어로 설정한 경우, IP 기반 감지는 잘못된 언어를 제공한다. `Accept-Language`는 사용자의 명시적 선호다.

## 번역

### DeepL Free API

- 무료 한도: 월 50만 자
- API Key: `DEEPL_API_KEY` 환경변수
- 엔드포인트: `https://api-free.deepl.com/v2/translate`
- source_lang: KO, target_lang: EN

### 번역 워크플로우 — 블로그 글

1. 관리자가 한국어로 글 작성/수정
2. "번역" 버튼 클릭 → `/api/translate` 호출 → DeepL API
3. 영어 번역 결과가 `title_en`, `content_en`, `excerpt_en`에 저장
4. 영어 탭에서 수동 교정 가능
5. 저장 시 한국어 + 영어 동시 저장

### 번역 워크플로우 — 포트폴리오, 카테고리, 시리즈

- 포트폴리오: 관리자 페이지(`/admin/portfolio`)에서 영어 필드 수동 입력
- 카테고리/시리즈: 관리자 페이지(`/admin/categories`)에서 영어명 수동 입력
- 항목 수가 적으므로 DeepL 자동 번역 불필요, 수동 입력이 더 정확

## 라우트 구조 변경

### Before

```
src/app/(public)/page.tsx
src/app/(public)/blog/page.tsx
src/app/(public)/blog/[slug]/page.tsx
...
```

### After

```
src/app/[lang]/(public)/page.tsx
src/app/[lang]/(public)/blog/page.tsx
src/app/[lang]/(public)/blog/[slug]/page.tsx
...
```

- `[lang]` 파라미터: `"ko"` | `"en"`
- 페이지는 SSR 유지 (블로그 글은 동적 콘텐츠이므로 정적 생성 불필요)
- 관리자 라우트(`/admin/...`)는 변경 없음 (한국어 전용)

## 미들웨어

### 기존

- `/admin/:path*` 매칭 → 인증 체크

### 변경

- 루트(`/`) 접속 → 언어 감지 → `/ko` 또는 `/en` 리다이렉트
- `/ko/...`, `/en/...` → 정상 통과
- `/admin/:path*` → 기존 인증 로직 유지

## 정적 텍스트 번역

### 파일: `src/lib/i18n/messages.ts`

```
{
  ko: {
    nav: { home: "홈", blog: "블로그", portfolio: "포트폴리오", about: "소개" },
    blog: { readMore: "더 읽기", search: "검색...", ... },
    comment: { name: "이름", password: "비밀번호", ... },
    ...
  },
  en: {
    nav: { home: "Home", blog: "Blog", portfolio: "Portfolio", about: "About" },
    blog: { readMore: "Read more", search: "Search...", ... },
    comment: { name: "Name", password: "Password", ... },
    ...
  }
}
```

### 유틸리티: `src/lib/i18n/utils.ts`

- `getMessages(lang)` — 해당 언어 메시지 반환
- `type Lang = "ko" | "en"`
- `SUPPORTED_LANGS`, `DEFAULT_LANG` 상수

## API 라우트

### 새 라우트

| Route | Method | 역할 |
|-------|--------|------|
| /api/translate | POST | DeepL API로 텍스트 번역 (관리자 전용, ADMIN_USER_ID 검증) |

### `/api/translate` 인증

- Supabase 세션에서 `auth.uid()` 확인
- `ADMIN_USER_ID`와 일치하지 않으면 403 반환
- DeepL 무료 한도 보호를 위해 인증 필수

### 기존 라우트 변경

- `/api/comments` — 변경 없음 (댓글은 언어 구분 없이 공유, lang 파라미터 불필요)

## SEO

- `<html lang="ko">` / `<html lang="en">` — `[lang]` 파라미터 기반
- `<link rel="alternate" hreflang="ko" href="/ko/..." />`
- `<link rel="alternate" hreflang="en" href="/en/..." />`
- `<link rel="alternate" hreflang="x-default" href="/ko/..." />` (기본 언어)
- sitemap.ts 업데이트: 각 페이지의 ko/en URL 모두 포함
